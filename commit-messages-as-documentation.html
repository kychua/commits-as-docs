Commit messages as documentation

  • summarise changes

  • provide context: when, who, why

  • don’t become outdated

  • searchable

Search

  • commits where code change contains the string

    • git log -S"diff contents"

  • commits where message contains the string

    • git log --grep="commit contents"

  • commits for given path or file

    • git log — path/or/file

  • show all the lines of a file and their current commit

    • git blame FILENAME

Let’s say one day, you used git blame and discovered the commit that caused the bug…​

…​and discovered the commit was 1000 lines long, including 3 new features

…​not that useful, is it?

Atomic

Only one logical change

  • whitespace fixes, refactorings separate from functional changes

  • different features

Atomic commits are easier to…​

  • review, debug, revert

Coherent

Commits should not break build

  • test updates in the same commit

  • code works as is

Coherent commits

  • allow you to use git bisect

Commit messages

  • summary of code changes

  • provide context behind the code change

  • make sense without seeing code diffs & external resources

  • consistent, correct spelling

  • correct formatting for readability

Summary

Explain what the commit does

  • headlines: a summary

  • make clear which part of code is affected

    • use type/scope: fix(DevGuide): add missing blank line

  • How does the commit solve the issue?

Why is the change needed?

  • fix bug

  • improve performance

  • enhance feature

  • Redirect user to the requested page after login

    • Users were being redirected to the home page after login, which is less useful than redirecting to the page they had originally requested before being redirected to the login form.

  • Set up AppVeyor CI

    • AppVeyor[1] is a continuous integration service used to build and test projects on a Windows VM. Adding AppVeyor support to this project will allow us to easily ensure that our tests do not break on Windows.

How does the commit solve the issue?

  • high level overview of code changes

  • Fix bug that occurs when latest commit is authored using git-pair

    • This commit fixes this bug by checking if the required commit author information is available first before trying to access it.

Why did you choose this approach?

  • tradeoffs

  • alternatives

    • difficulties faced

What are the side effectst?

  • impact on performance

  • impact on other features

  • breaks build?

What are the limitations?

  • known limitations

  • future scope for improvement

  • Set Up AppVeyor CI

    • For now, we will run our tests with 64-bit Java only, since the project does not use any native libraries and thus is unlikely to have any architecture-specific bugs. In the future, should we use any native libraries, the appveyor.yml file should be updated to run our tests on both 32-bit and 64-bit Java.

Write better commits

Good commits & useful commit messages

  • make it easier to stay updated on changes in the project

  • makes debugging easier by documenting the context of code changes

  • improves quality of your code by forcing you to think through your decisions
